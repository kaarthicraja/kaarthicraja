# 🏗️ Architecture & System Documentation

## Overview

This repository is the source of truth for the **kaarthicraja** GitHub profile. It is structured as a fully modular, self-updating system where static and dynamically generated assets are cleanly separated.

---

## 🔄 Automation Workflows

### 1. `quote.yml` — Daily Quote Generator

| Property | Value |
|----------|-------|
| Trigger  | Daily `cron: '0 0 * * *'` + `workflow_dispatch` |
| Auth     | `GITHUB_TOKEN` (built-in, no secrets needed) |
| Output   | `assets/generated/quote.svg` |
| Commit   | Bot commits with `[skip ci]` to prevent recursion |

**Flow:** Cron fires → Node 18 setup → `node scripts/generateQuote.js` → reads `quotes.json` → picks random quote → generates styled SVG → if changed, bot commits and pushes.

---

### 2. `snake.yml` — Contribution Grid Snake

| Property | Value |
|----------|-------|
| Trigger  | Push to `main` + daily `cron: '0 1 * * *'` + `workflow_dispatch` |
| Action   | `Platane/snk@v3` (official snake generator) |
| Auth     | `GITHUB_TOKEN` |
| Output   | `assets/generated/github-contribution-grid-snake.svg` (light) + `...-dark.svg` |
| Commit   | `EndBug/add-and-commit@v9` with bot author |

**Flow:** Trigger fires → `Platane/snk@v3` reads your public contribution calendar from GitHub API → renders snake path SVG → commits both light and dark variants back to `main`.

---

### 3. `metrics.yml` — Advanced GitHub Metrics

| Property | Value |
|----------|-------|
| Trigger  | Push to `main` + daily `cron: '0 2 * * *'` + `workflow_dispatch` |
| Action   | `lowlighter/metrics@latest` |
| Auth     | `GITHUB_TOKEN` (public data only) |
| Output   | `assets/generated/metrics.svg` |
| Plugins  | languages, activity, habits |

**Flow:** Trigger fires → `lowlighter/metrics` reads your public GitHub profile data → renders a comprehensive SVG dashboard showing top languages, recent activity, and commit habits → commits `metrics.svg` back to `main`.

> **Note:** To unlock private repo data or additional plugins (sponsors, etc.), create a PAT with `read:user` + `repo` scopes and store it as a repo secret named `METRICS_TOKEN`, then replace `${{ secrets.GITHUB_TOKEN }}` in `metrics.yml`.

## 🎨 SVG Animation System

All SVGs in `assets/animations/` use **SMIL (Synchronized Multimedia Integration Language)** animations — the native SVG animation standard. This ensures:

- ✅ **GitHub compatible** — GitHub's Markdown renderer allows SMIL but strips JavaScript
- ✅ **Zero dependencies** — no JS runtime, no CSS frameworks
- ✅ **Lightweight** — each file is under 15KB

## 🧩 Modularity Principle

Each component is designed to be **independently replaceable**:

- To swap the banner: replace `assets/animations/banner.svg`
- To add a new quote: add an object to `quotes.json`
- To change the color scheme: find/replace `#00f3ff` (cyan) and `#bb9af7` (purple) across `assets/animations/`
- To add a new icon: drop an SVG into `assets/icons/` and reference it in `README.md`

The `assets/generated/` folder should **never be manually edited** — its contents are always overwritten by the automation workflows.
